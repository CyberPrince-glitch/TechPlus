üíª Manual Installation

Step-by-step manual setup

Prerequisites

  • Python 3.11+
  • Node.js 18+
  • MongoDB 7.0+
  • Git

Installation Steps

# Clone repository
git clone https://github.com/your-repo/techpulse-ai.git
cd techpulse-ai

# Backend setup
cd backend
pip install -r requirements.txt

# Frontend setup
cd ../frontend
npm install
# or
yarn install

# Start MongoDB (if not running)
mongod --dbpath /path/to/your/db

# Start backend
cd ../backend
uvicorn server:app --host 0.0.0.0 --port 8001

# Start frontend (new terminal)
cd frontend
npm start

üêß Linux/Fedora Installation

Installation guide for Linux systems

# Install dependencies (Fedora)
sudo dnf install python3.11 nodejs npm mongodb-server git

# Or Ubuntu/Debian
sudo apt install python3.11 nodejs npm mongodb git

# Clone and setup
git clone https://github.com/your-repo/techpulse-ai.git
cd techpulse-ai

# Backend
cd backend
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Frontend
cd ../frontend
npm install

# Start services
systemctl start mongod
cd ../backend && python server.py &
cd ../frontend && npm start

Troubleshooting Guide

üîß Common Issues

Backend Not Starting

Problem:

ModuleNotFoundError or import errors

Solution:

# Reinstall dependencies
cd backend
pip install -r requirements.txt

# Check Python version
python --version  # Should be 3.11+

# Alternative: Use virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt

Frontend Not Loading

Problem:

React app not starting or showing errors

Solution:

# Clear cache and reinstall
cd frontend
rm -rf node_modules package-lock.json
npm install
# or
yarn install

# Check Node.js version
node --version  # Should be 18+

# Start development server
npm start

MongoDB Connection Error

Problem:

Cannot connect to MongoDB

Solution:

# Windows: Start MongoDB service
net start MongoDB

# Linux: Start MongoDB
sudo systemctl start mongod
sudo systemctl enable mongod

# Check MongoDB status
sudo systemctl status mongod

# Alternative: Use Docker
docker run -d -p 27017:27017 --name mongodb mongo:7.0

AI Content Generation Failing

Problem:

API key errors or generation failures

Solution:

  • Check if Emergent LLM key is properly set in backend/.env
  • Add alternative API keys (OpenAI, Gemini, Claude) in Admin Settings
  • Test API keys using the "Test" button in Admin panel
  • Check API usage limits and quotas
  • Verify internet connection for API calls

API Documentation

TechPulse AI provides a comprehensive REST API for all operations. Base URL: http://localhost:8001/api

üîê Authentication

POST /auth/login

Login and get access token

curl -X POST "http://localhost:8001/api/auth/login" \
     -H "Content-Type: application/json" \
     -d '{"username": "admin", "password": "admin123!@#TechPulse"}'
POST /auth/create-admin

Create default admin user

curl -X POST "http://localhost:8001/api/auth/create-admin"

üì° RSS Management

GET /feeds

Get all RSS feeds

POST /feeds/initialize

Initialize default RSS feeds (Admin only)

curl -X POST "http://localhost:8001/api/feeds/initialize" \
     -H "Authorization: Bearer YOUR_TOKEN"
POST /articles/collect

Collect articles from RSS feeds (Admin only)

curl -X POST "http://localhost:8001/api/articles/collect" \
     -H "Authorization: Bearer YOUR_TOKEN"

ü§ñ AI Content Generation

POST /generate

Generate AI content from articles

curl -X POST "http://localhost:8001/api/generate" \
     -H "Content-Type: application/json" \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -d '{
       "topics": ["AI", "Technology"],
       "language": "english",
       "tone": "professional",
       "length": "medium",
       "article_count": 3,
       "include_seo": true
     }'
GET /content

Get generated content

curl "http://localhost:8001/api/content?limit=10&language=english"
